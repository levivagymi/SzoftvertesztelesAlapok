Egy pizzáról tároljuk a nevét (String), méretét cm-ben (24, 32, 50) és bruttó árát (int, Ft)! Legyen lehetőség a nevét módosítani! Helytelen méret esetén PizzaMeretException, helytelen ár esetén PizzaArException keletkezzen (mindkettő checked)! "Egy pizzáról tároljuk a nevét, méretét (24, 32, 50 cm), bruttó árát! Legyen lehetőség a nevét módosítani!"

"Hozzunk létre 4 Pizza objektumot (2 érvényes, 2 érvénytelen)! Az érvényeseket tároljuk listában!"

A @BeforeAll-ban try-catch blokkokkal hozzuk létre az objektumokat. Érvényes adat → bekerül a listába. Érvénytelen adat → kivétel dobódik, az objektum null marad. Nincs külön assertNull/assertNotNull teszt — a listára és a getterekre támaszkodunk.

Tesztek:
@BeforeAll — nem @Test!nincs direkt teszteset, ez csak setup — az objektumokat a getter/setter/toString tesztek használják01
private static Pizza p1, p2, p3, p4;
private static ArrayList<Pizza> pizzak;

@BeforeAll
static void beforeAll() {
    pizzak = new ArrayList<>();
    // érvényes → listába kerül
    try { p1 = new Pizza("Pizza",   24, 2500); pizzak.add(p1); }
    catch (PizzaMeretException pme) { } catch (PizzaArException pae) { }

    try { p4 = new Pizza("Szalami", 50, 6000); pizzak.add(p4); }
    catch (PizzaMeretException pme) { } catch (PizzaArException pae) { }

    // érvénytelen → kivétel dobódik, változó null marad, listába nem kerül
    try { p2 = new Pizza("X", 900, 300);   pizzak.add(p2); } // rossz méret
    catch (PizzaMeretException pme) { } catch (PizzaArException pae) { }

    try { p3 = new Pizza("Y", 32, -2000);  pizzak.add(p3); } // negatív ár
    catch (PizzaMeretException pme) { } catch (PizzaArException pae) { }
}

"Az atalakit() metódus a szöveget 10 karakterenként sortöréssel egészítse ki!"

Határérték-analízis: ≤ 10 karakter → nincs sortörés, > 10 karakter → van. Fontos: a @BeforeAll-ban minden tesztesethez külön objektumot hozzunk létre, hogy ne befolyásolják egymást!

"Valósítson meg másoló konstruktort: a megnevezés az eredeti névből és az '_masolat' utótagból álljon!"

Másoló konstruktor → egy tesztmetódusban ellenőrizzük az összes adattagot (4 assertEquals egymás után). Nincs checked exception → a @BeforeAll-ban try-catch nélkül példányosítunk.

private static Butor b1, b2;

@BeforeAll
static void beforeAll() {
    // Nincs checked exception → try-catch nélkül!
    b1 = new Butor(1, 1, 1, "Asztal");
    b2 = new Butor(b1); // másoló konstruktor
}

@Test

    Assertions.assertEquals("Asztal_masolat", b2.getMegnevezes(), "Hiba a másoló konstruktor névbeállításakor!");
    Assertions.assertEquals(1, b2.getSzelesseg(), "Hiba a másoló konstruktor szélességbeállításakor!");
    Assertions.assertEquals(1, b2.getMagassag(),  "Hiba a másoló konstruktor magasságbeállításakor!");
    Assertions.assertEquals(1, b2.getMelyseg(),   "Hiba a másoló konstruktor mélységbeállításakor!");
}
"Negatív méretérték esetén IllegalArgumentException keletkezzen!"

IllegalArgumentException = unchecked exception → @BeforeAll-ban nem kell try-catch. A teszteléshez itt a B-mintát alkalmazzuk: return a catch-ben, fail() a try-catch után — szemantikailag pontosabb, mint az A-minta.

Tesztek:
negativKonstruktorTesztB-minta: return a catch-ben, fail() a try-catch után02
@Test
void negativKonstruktorTeszt() {
    try {
        Butor b = new Butor(-1, -1, -1, "Asztal");
    } catch (IllegalArgumentException iae) {
        return; // B-minta: kivétel keletkezett → teszt sikeres, visszatérünk
    }
    fail("Nem keletkezett IllegalArgumentException negatív méretek esetén!");
}

// tippek

Vizsgatippek

  • 💡 toString() gyors megírása: Először üres Stringgel írd meg a tesztet, futtasd le, és a megbukott teszt kimenetéből másold ki az actual értéket — illeszd be az assertEquals-ba. Nem kell kézzel begépelni.
  • 📊 Figyelj a pontozásra: Valószínűleg ahány pont van egy feladatnál, annyi teszteset kell — érdemes ebből visszaszámolni, hogy mit várnak el.
  • 📋 Elemezd először a specifikációt: Mielőtt nekiállsz kódolni, olvasd végig mit kell tesztelni — dob-e kivételt, double-e az érték, van-e tömb. Utána válaszd ki a megfelelő törvényt.
  • ⚠️ Ha a konstruktor kivételt dob, az objektum null marad: Mielőtt gettereket vagy settereket tesztelsz, bizonyosodj meg róla hogy az objektum tényleg létrejött a @BeforeAll-ban — különben NullPointerException-t kapsz futásidőben.
  • 🔢 @Order számokban hagyj rést: Ne 1, 2, 3... hanem 1, 4, 7, 9... sorrendet használj — így ha utólag közé kell illeszteni egy tesztet, nem kell az összes számot átírni.
  • 🔍 Checked vagy unchecked exception? Ha a konstruktor nem dob checked exceptiont (pl. Butor), a @BeforeAll-ban nem kell try-catch — egyenesen példányosítasz. Csak checked exception esetén kell a try-catch blokk.
  • ✏️ A fail() üzenete legyen informatív: Ne csak "hiba" — írd bele mi a probléma és mikor keletkezik, pl. "Hiba a setMeret() működésében, érvényes értéknél kivétel keletkezett!". Vizsgán ez mutatja hogy érted a tesztet.
